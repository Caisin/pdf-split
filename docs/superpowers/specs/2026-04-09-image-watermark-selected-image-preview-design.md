# 图片水印单图预览设计

## 背景
现有批量图片水印工具已经支持从输入目录选图，但前端叠层模拟的预览不够真实。用户希望看到和后端实际水印逻辑一致的真实预览结果。

## 目标
在不改变批量处理主命令的前提下，支持从输入目录递归枚举可预览图片，并基于选中的单张图片生成后端真实预览，参数变化时自动刷新。

## 方案
- 新增轻量 Tauri 命令：
  - 列出输入目录下可预览图片（递归，返回相对路径）
  - 基于指定图片和当前参数生成真实预览图片字节
- 前端在选择输入目录后拉取图片列表，并默认选中第一张。
- 前端在参数变化后以 400ms 防抖自动请求真实预览。
- 当新预览生成中时，继续保留上一张预览图，并在角落显示“更新中”状态。
- 如果旧请求晚于新请求返回，前端直接丢弃旧结果，只展示最后一次参数对应的预览图。
- 预览区显示后端返回的真实结果图，不再以前端叠层模拟作为主预览。
- 当目录没有图片、预览生成失败或尚未选择目录时，显示占位提示。

## 边界
- 不改 `add_text_watermark_to_images` 的 payload 和处理逻辑。
- 不新增依赖。
- 真实预览与批量处理使用同一套后端水印逻辑，但预览只处理单张图。

## 测试策略
- 前端测试覆盖目录选图选择器出现、真实预览生成与 400ms 防抖刷新。
- Rust 测试覆盖非法路径校验与图片列表过滤逻辑。
