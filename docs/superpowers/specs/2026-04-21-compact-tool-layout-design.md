# 紧凑工具界面布局设计

## 目标
把以下 3 个工具界面的组件大小与排版统一到接近“批量图片水印”的紧凑风格，减少每个组件独占一整行带来的纵向空间浪费：

- PDF 转图片
- 提取 PDF 内嵌图片
- 批量 PDF 文字水印

## 非目标
- 不新增功能
- 不改变现有命令调用与业务逻辑
- 不新增预览区域
- 不改顶部 Tab 结构

## 设计原则
1. **统一紧凑卡片风格**：上述 3 个页面统一采用 `tool-card-dense` 的卡片密度。
2. **优先横向排布**：文件/目录选择器优先进入 `picker-grid` 两列布局。
3. **短字段并排，长字段独占**：短输入项进入 `field-grid`，文本域继续整行展示。
4. **与批量图片水印对齐**：组件高度、间距、按钮、状态行尽量沿用现有紧凑样式，而不是为每个工具页单独定制一套规则。
5. **小改结构，不加新抽象**：优先调整现有 JSX 结构和已有 CSS class 组合，不新增复杂通用组件层。

## 页面级布局

### 1. PDF 转图片
- 卡片切换为紧凑样式。
- `PDF 文件` 与 `输出目录` 放入 `picker-grid`。
- `图片格式` 作为短字段，与其余短字段风格保持一致。
- 提交按钮与状态行保持当前语义，但沿用紧凑间距。

### 2. 提取 PDF 内嵌图片
- 卡片切换为紧凑样式。
- `PDF 文件` 与 `输出目录` 放入 `picker-grid`。
- 因页面字段较少，不额外增加新说明块，只缩短纵向空白。

### 3. 批量 PDF 文字水印
- 保持当前“目录到目录”的批处理语义。
- `输入目录` 与 `输出目录` 放入 `picker-grid`。
- `水印字号` 作为短字段，不再单独占据完整纵向区块。
- `水印文字` 保留整行文本域。
- 目录冲突提示、按钮、状态行延续当前逻辑，仅收紧布局。

## CSS 调整范围
- 复用现有：`tool-card-dense`、`picker-grid`、`field-grid`
- 必要时仅做轻量补充：
  - 为 checkbox / 较短表单项补一个紧凑对齐样式
  - 保持响应式下可回落为单列

## 验收标准
1. 上述 3 个工具页不再主要依赖“一项一行”的纵向堆叠。
2. 与“批量图片水印”相比，卡片密度、输入高度、间距明显接近。
3. 现有测试和构建继续通过。
4. 不引入功能回归。

## 风险与控制
- **风险**：紧凑化后，窄屏下字段可能拥挤。
- **控制**：保留现有响应式回退，让较窄宽度下自动转回单列。
