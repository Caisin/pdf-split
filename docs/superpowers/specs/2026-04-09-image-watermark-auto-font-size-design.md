# 图片水印自动字号设计

## 背景
固定像素字号在大图上会显得过小，导致真实预览与批量输出都难以达到稳定可读的效果。

## 目标
将固定字号改成“可调长边比例系数 + 后端自动换算”，并让正式批量处理与真实预览保持一致。

## 规则
- 基础字号：按图片长边比例计算。
- 前端提供可调的**长边字号比例 (%)**，默认 `2.8%`。
- 最小字号：不小于图片短边的 `1%`。
- 最大字号：不大于图片短边的 `50%`。
- 最终字号：`clamp(long_edge * 用户比例, short_edge * 0.01, short_edge * 0.5)`。
- 横向间距：按图片宽度比例计算。
- 纵向间距：按图片高度比例计算。

## 方案
- 前端提供以下可调比例参数：
  - 长边字号比例 (%)
  - 横向间距比例 (%)
  - 纵向间距比例 (%)
- Tauri 命令层不再接收固定像素字号/间距，统一接收比例值。
- `kx-image` 在单图渲染前根据当前图片宽高动态计算真实字号和真实间距。
- 真实预览与批量处理复用同一套比例换算逻辑。

## 测试策略
- 前端测试覆盖：比例输入渲染、预览/批处理 payload 传比例字段。
- Rust 测试覆盖：自动字号计算遵守长边比例与短边 1%-50% 钳制规则，图片水印批处理仍保持可用。
