# 批量图片文字水印设计文档

日期：2026-04-08
项目：`pdf-split`

## 1. 目标

在当前 Tauri 应用中新增一个“批量图片文字水印”工具页，支持：

1. 选择一个输入目录，递归扫描其中所有子目录图片。
2. 选择一个输出目录，并按输入目录的相对结构输出处理结果。
3. 对图片批量添加“平铺斜向重复”的文字水印。
4. 输入目录与输出目录不能相同。
5. 保持原文件名与原图片格式。
6. 单个文件失败时跳过并继续，最终汇总成功数与失败数。

## 2. 已确认需求

### 2.1 输入输出规则

- 用户选择输入目录，而不是单文件或多文件。
- 需要递归处理子目录中的图片。
- 输出目录保留输入目录的相对结构。
- 输入目录与输出目录不能相同。
- 为避免递归扫描输出结果，输出目录也不允许位于输入目录内部。

### 2.2 水印样式

- 水印类型：文字水印。
- 排版方式：平铺斜向重复。
- 允许在界面中调整：
  - 水印文字
  - 字号
  - 透明度
  - 旋转角度
  - 横向间距
  - 纵向间距

### 2.3 输出规则

- 保持原图片格式。
- 保持原文件名，不追加后缀。
- 如果输出目标文件已存在，将该文件记为失败，不中断整个批处理。

### 2.4 失败处理

- 损坏图片、无权限文件、无法解码文件、目标文件冲突：跳过并继续。
- 最终展示：扫描图片数、成功数、失败数、输出目录。

## 3. 职责边界

### 3.1 当前应用前端负责

- 新增“批量图片水印”页签与表单。
- 选择输入目录与输出目录。
- 收集水印参数。
- 做轻量前端校验并显示状态。
- 调用 Tauri 命令并展示结果汇总。

### 3.2 当前应用 Rust 命令层负责

- 校验输入输出目录与参数。
- 调用底层图像批处理能力。
- 将处理结果映射成前端可消费的结构。

### 3.3 `kx-pdf` 本地依赖负责

- 递归遍历图片文件。
- 识别支持的静态图片格式。
- 生成平铺斜向重复的文字水印图层。
- 将水印图层叠加到原图并保存到输出目录。
- 汇总成功/失败结果。

## 4. 技术方案

采用与现有 PDF 工具一致的结构：

- React 页面收集参数并触发命令。
- Tauri Rust command 保持轻量。
- 真实图片处理逻辑下沉到本地 `kx-pdf`。

这样可以复用现有 PDF 水印能力中的字体定位、文字栅格化、旋转与图层叠加逻辑，并让当前项目继续保持“桌面编排层”的角色。

## 5. 界面设计

新增第四个顶部页签：`批量图片水印`。

表单字段：

- 输入目录
- 输出目录
- 水印文字
- 水印字号
- 透明度（建议 1-100）
- 旋转角度（度）
- 横向间距（像素）
- 纵向间距（像素）

交互规则：

- 未选择输入目录、未选择输出目录、文字为空、参数非法时禁用提交按钮。
- 输入目录与输出目录相同时禁用提交按钮并显示提示。
- 执行中显示“处理中...”。
- 成功后显示汇总信息。
- 失败时显示命令返回的错误信息。

## 6. 后端接口设计

新增 Tauri 命令：`add_text_watermark_to_images`

输入：

- `input_dir: String`
- `output_dir: String`
- `watermark_text: String`
- `watermark_font_size: f32`
- `watermark_opacity: f32`
- `watermark_rotation: f32`
- `watermark_horizontal_spacing: u32`
- `watermark_vertical_spacing: u32`

输出：

- `scanned_file_count: usize`
- `success_count: usize`
- `failure_count: usize`
- `output_dir: String`

## 7. 水印渲染规则

- 使用固定灰色文字。
- 文本先栅格化到透明图层。
- 按旋转角度生成单个斜向水印 stamp。
- 以 `stamp 宽高 + 间距` 的网格平铺到整张图。
- 奇偶行进行半步错位，使视觉更接近连续斜向覆盖。
- 再将图层叠加到原图。

## 8. 图片范围与格式策略

第一版支持常见静态图片格式：

- `png`
- `jpg`
- `jpeg`
- `webp`
- `bmp`
- `tif`
- `tiff`

非支持格式直接忽略，不计入图片扫描数。

说明：

- 第一版不处理动画 GIF。
- 第一版不保留 EXIF/元数据搬运承诺，只保证像素结果和文件格式输出。

## 9. 错误处理

整体错误：

- 输入目录不存在
- 输出目录与输入目录相同
- 输出目录位于输入目录内部
- 没有找到可处理图片
- 字体文件不可用
- 参数非法

单文件错误：

- 图片解码失败
- 输出目录创建失败
- 文件写出失败
- 目标文件已存在

单文件错误只增加失败计数，不中断整个任务。

## 10. 测试策略

### 10.1 前端测试

- 新页签存在。
- 新页签默认提交按钮禁用。
- 输入输出目录相同时提交按钮禁用并显示提示。
- 参数控件正常渲染。

### 10.2 当前项目 Rust 测试

- 新命令拒绝空输入目录。
- 新命令拒绝相同输入输出目录。
- 新命令拒绝非法透明度或非法间距。

### 10.3 `kx-pdf` 测试

- 递归处理子目录图片并保留结构。
- 输出图片成功生成。
- 同目录输入输出被拒绝。
- 单文件失败不会阻断其它图片成功输出。
